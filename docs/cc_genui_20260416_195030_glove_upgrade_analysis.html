The Problem: Why GloVe Failed

Observed Failure Mode

All 12 philosophical faces returned coordinates (0, 0) with 0.4 weight and 0.0 coherence. This is a known domain-specific ceiling — not a bug in the bridge logic.

Three root causes compound:

CauseExplanation
Static vectors GloVe assigns one vector per word regardless of context. "Architecture" (building) and "architecture" (software) are identical.
No compositional semantics Multi-word concepts like "hierarchical coordination" or "behavioral schema" have no single vector. Averaging sub-word vectors dilutes meaning to the centroid.
Training corpus mismatch GloVe was trained on Wikipedia/Gigaword. Software architecture vocabulary is sparse in those corpora, so domain terms cluster near generic regions of the space.

Tier 2: Domain Adaptation

Precision If Tier 1 moves the needle but doesn't fully resolve domain coherence, adapt the model to your specific vocabulary.

Option A: Contrastive Fine-tuning

  • Take a Tier 1 model (e.g., BGE-large) and fine-tune on pairs of related/unrelated architecture concepts
  • SBERT supports this directly with InputExample pairs and contrastive loss
  • A few thousand pairs (drawn from your own documentation, specs, ADRs) can dramatically improve in-domain coherence
  • Example pair: ("behavioral schema processing", "schema-driven code generation") = related; ("behavioral schema processing", "CSS grid layout") = unrelated

Option B: Instruction-tuned Models

  • instructor-xl: Prefix queries with task descriptions — "Represent this software architecture concept for semantic similarity"
  • nomic-embed-text-v1.5: Supports task-type prefixes (search_document:, search_query:)
  • Often beats fine-tuning for small-data cases where you don't have thousands of labeled pairs

Effort vs. Payoff

Fine-tuning requires curating concept pairs and running training. Only pursue this if Tier 1 still shows domain-ceiling behavior (concepts clustering too tightly or coordinates lacking discriminative spread). The instruction-tuned approach is lower effort and may suffice.

The Projection Layer: Fixing the Coordinate Space

All Tiers The embedding is only one half. The coordinate projection matters equally.

The current bridge maps embeddings to a 2D coordinate space where all concepts collapsed to the origin. Even with better embeddings, a naive projection can reproduce this. The fix is structural:

TechniqueWhat It DoesWhy It Helps
UMAP Non-linear dimensionality reduction preserving local + global structure Maintains cluster separations that PCA/linear projections collapse
PaCMAP Pairwise-Controlled Manifold Approximation Better global structure preservation than UMAP; fewer hyperparameters
Anchor concepts Hand-place 4-8 reference concepts at known coordinates Gives the space semantic orientation — "center" and "edge" become meaningful

Recommended Combination

Domain-adapted embeddings + UMAP/PaCMAP + anchor concepts eliminates the "all clustered at origin" failure mode. The anchors give the coordinate space a semantic frame of reference, and the non-linear projection preserves the structure that the embeddings encode.
